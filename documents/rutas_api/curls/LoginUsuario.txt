2. LOGIN DE USUARIO

Descripción:
Este comando prueba el endpoint de autenticación tradicional (/users/auth/login). Envía las credenciales registradas previamente para obtener un token JWT válido que se usará en futuras peticiones protegidas.

Comando:

curl -X POST http://localhost:3001/users/auth/login \

-H "Content-Type: application/json" \

-d '{

  "email": "matias@example.com",

  "password": "123456"

}'
